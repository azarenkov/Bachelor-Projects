## Notes on outliers

- **`YulMath.sol` lines 63%.** The `sqrt` seeding cascade has six nested `if`
  guards that progressively shift the input to estimate the magnitude of the
  result. Exercising every distinct path requires a value in each magnitude
  band (covered for many but not all). The arithmetic correctness is
  property-tested in `test/fuzz/YulMath.fuzz.t.sol` against arbitrary inputs.
- **`SimpleAMM` branches 63%.** The `addLiquidity` branch matrix has six
  combinations of `(amount0Desired vs amount1Desired vs reserves)` plus their
  slippage paths — covering all of them is left as a follow-up; the happy
  paths and revert paths are covered.
- **`TreasuryV1`/`V2` branches 50%.** The OZ `_authorizeUpgrade` is empty (no
  branches), and the per-token cap conditional has one branch with cap=0
  (skipped) and one with cap>0 (enforced) — both are exercised, but the
  coverage tool counts the cap==0 short-circuit and the `to == address(0)`
  guard as one branch pair each.
